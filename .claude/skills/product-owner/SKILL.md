# Datti Product Owner Skill

このスキルは、Linearを使ったプロダクト管理を支援します。

## 概要

Dattiでは Linear を使ってタスク管理を行っています。このスキルでは、チケットの作成・整理・優先度設定・説明の肉付けなど、プロダクトオーナー業務を支援します。

## Linear の基本情報

| 項目 | 値 |
|------|-----|
| チーム名 | Datti |
| チケット形式 | `DATTI-xxx` |
| GitHub連携 | PRタイトルに `DATTI-xxx` を含めると自動リンク |

## ラベル体系

| ラベル | 用途 |
|--------|------|
| Bug | バグ修正 |
| Feature | 新機能追加 |
| Improvement | 改善・リファクタリング |

## 優先度体系

**Priorityフィールドを使用**する（手動並び替えではなく）。

| Priority | 値 | 用途 |
|----------|-----|------|
| Urgent | 1 | 今すぐ対応が必要 |
| High | 2 | 早めに対応したい（バグ等） |
| Medium | 3 | 通常の機能開発・改善 |
| Low | 4 | 余裕があれば / 大きめの改修 |
| No priority | 0 | 未設定 / 保留 |

**優先度の目安**:
- **High**: バグ、ユーザー影響が大きい問題
- **Medium**: 近いうちにやりたい機能改善、開発体験向上
- **Low**: 将来的な機能追加、大規模リファクタリング、インフラ改善

## チケット作成

### 基本ルール

- **タイトル**: 何をするかが明確にわかるように
- **説明**: 背景・やりたいこと・必要な作業を記載
- **ラベル**: Bug / Feature / Improvement のいずれかを付与
- **優先度**: 適切な Priority を設定

### 説明のテンプレート

#### バグの場合

```markdown
## 現象

（何が起きているか、どの画面で発生するか）

## 再現手順（わかれば）

1. ...
2. ...

## 期待する動作

（本来どうあるべきか）

## 原因（推測）

（技術的な原因の仮説）
```

#### 機能追加の場合

```markdown
## やりたいこと

（何を実現したいか）

## 背景

（なぜこの機能が必要か）

## 仕様

（具体的な動作、計算ロジック等）

## UI案（あれば）

（画面イメージ、配置など）

## 必要な作業

- [ ] バックエンド: ...
- [ ] フロントエンド: ...

## 依存（あれば）

（先に実装が必要なチケット）
```

#### 改善・リファクタリングの場合

```markdown
## 背景

（現状の課題）

## やりたいこと

（どう改善するか）

## 実施内容

- [ ] ...
- [ ] ...

## 参考

（関連ドキュメント、既存の実装箇所など）
```

### タイトルのガイドライン

**良いタイトル**:
- 具体的な動作を記述（`イベント日が1日ズレる・更新後すぐ反映されない`）
- 何をするかが明確（`金額入力のスピンボタンを非表示にする`）
- 技術用語は括弧で補足（`DBスキーマ変更の承認フローを導入する（Bytebase）`）

**避けるべきタイトル**:
- 内部用語だけ（`Borrowingを廃止`）
- 略語だらけ（`IaCのTF→Plumi化`）
- 曖昧（`〜できない`、実際は別の問題）
- 長すぎる

## チケット整理

### 整理フロー

1. **現状把握**: 全チケットの状況を確認
2. **優先度設定**: Priority を一括設定
3. **説明の肉付け**: 空の説明を埋める
4. **ラベル整理**: 未設定のラベルを付与
5. **タイトル精査**: わかりにくいタイトルを修正

### ユーザーへのヒアリング

- 一度に聞きすぎない（3〜5件ずつ）
- 選択肢を提示する（「〜ですか？」より「A or B どちらですか？」）
- わかる範囲でOKと伝える
- 技術的な詳細は推測で補完し、確認を取る

## MCP コマンドリファレンス

### チケット操作

```
# 一覧取得
mcp__linear-server__list_issues(team: "Datti", limit: 100)

# 詳細取得
mcp__linear-server__get_issue(id: "DATTI-xxx")

# 作成
mcp__linear-server__create_issue(
  team: "Datti",
  title: "...",
  description: "...",
  labels: ["Feature"],
  priority: 3
)

# 更新
mcp__linear-server__update_issue(
  id: "DATTI-xxx",
  title: "...",
  description: "...",
  labels: ["Bug"],
  priority: 2
)
```

### マスタ情報

```
# ラベル一覧
mcp__linear-server__list_issue_labels(team: "Datti")

# ステータス一覧
mcp__linear-server__list_issue_statuses(team: "Datti")

# ユーザー一覧
mcp__linear-server__list_users(team: "Datti")
```

### 関連操作

```
# コメント追加
mcp__linear-server__create_comment(issueId: "...", body: "...")

# プロジェクト一覧
mcp__linear-server__list_projects(team: "Datti")
```

## 注意事項

- **ビュー設定はMCPから変更不可**: Ordering を Priority に変更する等は Linear UI で手動設定が必要
- **並列更新OK**: 複数チケットの更新は並列で実行して効率化
- **確認を取る**: 優先度やタイトル変更は一覧で提示し、ユーザーの承認を得てから実行
